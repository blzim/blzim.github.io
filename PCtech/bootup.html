Boot Up Process

Two stages: Getting the hardware running, then getting the Operating System running.

The power supply gets AC power and provides DC power to the system.

Power On Self Test (POST). POST checks that the hardware is working correctly. POST will either use a system of beep codes or a display on the motherboard if it runs into errors. The system manufacturer usually provides a table of POST code meanings.

Boot order occurs according to the BIOS parameters often called the System Setup, which is stored in what was called the Complimentary Metal Oxide Semiconductor (CMOS), now being more generally called the NVRAM, because some systems are moving from CMOS to flash NAND to store the BIOS settings.

Contol passes to the first block of the primary boot device. Hard drives have typically been formatted with 512 byte sector blocks. In 2009, Hard drives started moving to 4096 byte sectors known as Advanced Format (AF) or 4K drives. 4K sectors are 8 times the old sector size and prove to be much more efficient in larger capacity drives and has room for better error correction.

What was previously called the Master Boot Record (MBR), on some systems is now the GUID Partition Table (GPT), depending on how the drive was formatted. This is the same for HDDs, SSDs, and Flash NAND USB boot sticks. MBR works with the 512 byte HDD sectors and GPT is used for 4K drives.

The first block of the boot device contains a partition table and a boot loader program that points to the boot program of the selected Operating System. Windows is called the Windows Bootloader, Macs use the Darwin Bootloader, and Linux was called Linux Loader (Lilo), but has been replaced in newer systems by the Grand Unifited Bootloader (GRUB).

The Boot loader usually provides a menu allowing the user to select which Operating System to run on systems that have multiple OSes installed.

The Operating system start sequences might vary slightly. In Linux, GRUB loads the kernel core into RAM, then runs the initrd (initial ramdisk), which then runs the rest of the Kernel and OS including the real root filesystem, and the runlevel programs from the run command daemond (rc.d).

The GRUB configuration file shows the boot options including the device, the kernel filename (the compressed Linux kernel is called vmlinuz), and the initial ramdisk called initrd.
